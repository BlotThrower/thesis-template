\subsection{CNN-LSTM}
\begin{lstlisting}[language=Python, caption={PyTorch-Implementierung der hybriden CNN-LSTM Architektur}, label={lst:cnnlstm}]
class CNNLSTMClassifier(nn.Module):
    def __init__(self, output_dim, num_features):
        super(CNNLSTMClassifier, self).__init__()

        # CNN Block 1
        self.conv1 = nn.Conv1d(in_channels=num_features, 
            out_channels=32, kernel_size=5, padding=2)
        self.bn1 = nn.BatchNorm1d(32)
        self.pool1 = nn.MaxPool1d(kernel_size=2)

        # CNN Block 2
        self.conv2 = nn.Conv1d(in_channels=32, out_channels=64, 
            kernel_size=5, padding=2)
        self.bn2 = nn.BatchNorm1d(64)
        self.pool2 = nn.MaxPool1d(kernel_size=2)

        # LSTM Block
        self.lstm = nn.LSTM(input_size=64, hidden_size=128, 
            num_layers=2, dropout=0.3, batch_first=True)

        # Classification Head
        self.fc_combined = nn.Linear(128, 128)
        self.dropout = nn.Dropout(0.5)
        self.out = nn.Linear(128, output_dim)

    def forward(self, x_ts):
        # Prepare for 1D Convolution: (Batch, Channels, Time)
        x_ts = x_ts.permute(0, 2, 1)

        # Apply CNN Layers
        x_ts = self.pool1(F.relu(self.bn1(self.conv1(x_ts))))
        x_ts = self.pool2(F.relu(self.bn2(self.conv2(x_ts))))

        # Prepare for LSTM: (Batch, Time, Channels)
        x_ts = x_ts.permute(0, 2, 1)

        # Apply LSTM
        lstm_out, _ = self.lstm(x_ts)
        
        # Global Average Pooling across time dimension
        # Shape: (Batch, 128)
        ts_features = torch.mean(lstm_out, dim=1)

        # Apply Dense Layers
        x = F.relu(self.fc_combined(ts_features))
        x = self.dropout(x)
        return self.out(x)
\end{lstlisting}

Das in \ref{lst:cnnlstm} abgebildete CNN-LSTM-Hybridmodell besteht aus einem CNN mit zwei eindimensionalen Faltungsschichten. Diese Schichten bilden die eingehenden Datenkanäle erst auf 32 und dann auf 64 Feature-Maps. Danach folgt eine Batch-Normalisierung zur Stabilisierung des Trainingsprozesses und die ReLU-Aktivierungsfunktion. Es folgt eine Max-Pooling-Funktion mit Größe $2$. Diese dient zur Feature-Verdichtung und Maxima-Bestimmung, wodurch wichtigere Features deutlicher hervortreten \cite{Chollet2025}. Die zweite Faltungsschicht wiederholt den Prozess und erweitert die Feature-Maps auf 64 Kanäle.

Im Laufe der Untersuchungen haben sich Smartphonesensoren als Basis der Messreihen zur Klassifizierung von Bodenbelägen, sowie das gewählte Setup als vielversprechend und ausbaufähig erwiesen. Es wird jedoch noch weitere Arbeit und fortwährende Anpassungen sowohl im gewählten ML-Modell als auch in im Aufnahme-Setup notwendig sein.
